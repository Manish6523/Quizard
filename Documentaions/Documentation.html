AI Quiz App 📝

📖 Overview

This section provides a high-level understanding of the AI-powered quiz platform, its purpose, and core value proposition.

An AI-powered platform that allows educators to generate quizzes and tests using natural language prompts. The platform enables easy sharing via links, provides real-time scoring, AI-generated explanations, and performance analytics — all within a secure, user-friendly interface.

This platform bridges the gap between teachers and modern AI by allowing:

  • Quiz generation using AI prompts.
  • Manual editing of question sets.
  • Sharing quizzes with students via a link.
  • Timed quiz attempts and scoring.
  • Real-time analytics and feedback.

✨ Features

Discover the key functionalities that make this platform powerful and user-friendly for educators and students.

  • 🔮 AI Quiz Generator – Convert text prompts into structured quiz formats (MCQs, True/False, Short Answer).
  • ✏️ Quiz Editor – Edit AI-generated quizzes manually.
  • 🔗 Sharable Links – Generate quiz URLs with access controls.
  • ⏱️ Timer Support – Add quiz durations and enforce deadlines.
  • 📊 Live Analytics – Track quiz attempts, scores, and response accuracy.
  • 🧠 Bloom’s Taxonomy Levels – Choose difficulty (Knowledge, Application, Evaluation).
  • 📥 Export Results – Download quiz results as CSV or PDF.
  • 🧪 Auto Grading – Immediate feedback with explanations.

🔐 Authentication

Details on how user authentication is handled, ensuring secure access for teachers while providing seamless access for students.

Supabase provides email/password and third-party OAuth (Google) logins.

  • Protected routes ensure that only logged-in teachers can access dashboards.
  • Students do not need to log in — quiz links are access-controlled via query tokens or UUIDs.

🧪 AI Integration

This section details how the platform interacts with the AI model to generate quiz content, including input and output formats.

Prompt passed to OpenAI via `generateQuestions(prompt: string)`. Receives structured JSON format: 

[
  {
    "question": "What is the capital of France?",
    "options": ["Paris", "London", "Berlin", "Madrid"],
    "answer": "Paris",
    "explanation": "Paris is the capital and most populous city of France."
  }
]

You can control:

  • Question count
  • Difficulty
  • Type (MCQ, TF, Short Answer)

🏗️ High-Level Architecture

A visual representation of the application's structure and data flow, detailing how different components interact.

System Diagram

User Browser/Client
Next.js Frontend
Next.js API Routes
🤖 AI Model Provider
🗄️ Supabase Backend

Flow Description

  1. User Interaction: Users interact with the Next.js frontend to create prompts, manage quizzes, or take tests.
  2. Quiz Generation: The frontend sends a request to a Next.js API route.
  3. AI Integration: The API route makes a server-side call to the AI model, passing the user's prompt.
  4. Data Persistence: The AI model returns generated questions, which the API route stores in the Supabase database.
  5. Quiz Retrieval & Attempt: The frontend fetches quiz questions from Supabase for participants.
  6. Authentication: Supabase Auth handles user sign-up, login, and session management.
  7. Quiz Submission & Scoring: On completion, answers are sent to an API route, which updates the score in Supabase.

📋 Naming Conventions

Guidelines for consistent naming across the project's codebase and database, promoting readability and maintainability.

  • Files: PascalCase for components, camelCase for utils.
  • Variables: camelCase.
  • Functions: Verb-based (e.g., `generateQuestions`, `submitQuiz`).
  • Pages: Match route names (e.g., `quiz.tsx`, `dashboard.tsx`).
  • Database Tables: snake_case (e.g., `quiz_questions`).

⚙️ Setup Instructions

Follow these steps to get the project up and running for local development.

1. Clone the repository

git clone <https://github.com/your-username/ai-quiz-generator.git>
cd ai-quiz-generator

2. Install dependencies

npm install

3. Create and configure .env.local

Copy the example file and fill in your OpenAI, Supabase, and other keys.

cp .env.example .env.local

4. Run the development server

npm run dev

⚠️ Make sure your Supabase project has the necessary tables: `quizzes`, `questions`, `responses`, `users`.
